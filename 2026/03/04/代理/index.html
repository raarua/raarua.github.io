RAG 基本问题

RAG（Retrieval-Augmented Generation，检索增强生成）：
让大模型在回答前先查资料，再结合资料生成答案，减少瞎编。

  1. 分片（Chunking）

    • 把长文档切成很多「片段」（chunk），常见切法：
      • 按段落 / 小节
      • 按字数或 token 数，比如每 300～500 字一块
      • 记录来源信息：标题、页码、章节
    • 目标：每个片段信息够完整，又不会太长。

  2. 索引（Indexing）

    • 对每个片段做 embedding（向量化），得到一个高维向量。
    • 把「片段文本 + 向量 + 元信息」保存到向量数据库 / 知识库里。
    • 这一步相当于给每个片段一个「坐标」，方便后面按相似度快速查找。

  3. 召回（Retrieval / Recall）

    • 用户提问题，对问题做 embedding，得到问题向量。
    • 在向量库里，用向量相似度（余弦相似度等）找出若干个最相近的片段（例如 top 20）。
    • 这是「粗筛」阶段：从大量文档里捞出一小撮相关候选。

  4. 重排（Rerank）

    • 用 cross-encoder / reranker 这样的模型，对「问题 + 每个候选片段」重新打分、排序。
    • 只保留前几条（比如 top 5）作为最终「证据」。
    • 这是「精筛」阶段：把最有用的几段放在最前面。

  5. 生成（Generation）

    • 把「问题 + 若干高相关片段」一起输入大模型，让模型基于这些片段生成答案。
    • 可以要求模型：
      • 优先引用片段内容
      • 标出来源（第几条、哪一页）
      • 不知道就说不知道，减少幻想。

一句话总结：

RAG = 先按向量检索相关片段（R），再让大模型结合这些片段生成答案（G）。
