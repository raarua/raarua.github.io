agent skill是一个说明文档，用来教大模型“怎么做事”。 MCP 和 agent skill 的区别 MCP（Model Context Protocol） 更像是「给大模型接工具 / 接数据源」。 通过 MCP，可以让模型去访问数据库、HTTP 接口、本地文件等，把外部数据“拿进来”用。 重点是：提供能力 / 数据通道，类似「给模型接各种 API」。 agent skill 是一份纯文档（Markdown），写的是「在某个任务上，你应该按什么步骤做」。 内容包括：任务背景、注意事项、推荐步骤、示例回答等。 重点是：教模型怎么用现有能力来完成任务，类似「给模型写操作手册 / SOP」。 可以简单记： MCP = 插件 / 工具 / 数据源（解决“能做什么 / 从哪拿数据”） agent skill = 使用说明 / 工作手册（解决“拿到数据后要怎么处理、按什么流程做”）

定义REST API (representational state transfer)，表现层状态转移。通过REST API可以对数据的状态进行增删改查。在http协议中是一种常用的接口协议。 与普通接口协议的区别先看一种「不规范」的写法以前很多接口是这样设计的：一个动作一个地址，动作写在 URL 里。 123/adm/getDoc → 查文档/adm/postDoc → 新增文档/adm/deleteDoc → 删除文档 这样会有一大堆地址（getXxx、postXxx、deleteXxx…），难记、难维护。 用 REST 怎么改？把「动作」提前到 HTTP 方法上REST 的做法是：地址只表示「资源」（比如文档），动作用 HTTP 方法表示。 还是「文档」这个资源，可以只用一个资源地址，用不同请求方法区分增删改查： 请求方法 地址示例 含义 GET /adm/docs 查文档列表 GET /adm/docs/1 查 id=1 的文档 POST /adm/docs 新增一篇文档 PUT /adm/docs/1 修改 id=...
